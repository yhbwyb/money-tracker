# 记账本 — 部署与使用教程

## 概述

这是一个纯前端的手机记账 PWA 应用，部署到 GitHub Pages 之后，手机浏览器打开一次、添加到主屏幕，即可像原生 App 一样离线使用。数据全部存在手机本地，不上传任何服务器。

---

## 一、部署（约 5 分钟）

### 1. Fork 仓库

在 GitHub 网页打开 `https://github.com/yhbwyb/money-tracker`，点击右上角 **Fork** 按钮，Fork 到自己的 GitHub 账号下。

### 2. 开启 GitHub Pages

进入你 Fork 后的仓库 → **Settings** → **Pages**：

- **Source**：选择 `Deploy from a branch`
- **Branch**：选择 `gh-pages`，目录选 `/ (root)`
- 点击 **Save**

### 3. 等待首次部署

回到仓库主页 → **Actions** 标签页，等待 `Deploy to GitHub Pages` 工作流跑完（约 1-2 分钟）。看到绿色勾即部署成功。

### 4. 确认线上地址

部署成功后，线上地址是：

```
https://<你的GitHub用户名>.github.io/money-tracker/
```

电脑浏览器打开这个地址确认能正常加载（可能需要翻墙，GitHub Pages 在国内偶尔不稳定）。

## 二、在手机上安装 PWA（iPhone）

### 前提

- iPhone 连接到可访问 GitHub Pages 的网络（可能需要临时开翻墙工具）
- 使用 **Safari 浏览器**（微信内置浏览器不支持 PWA 安装）

### 步骤

1. **打开翻墙工具**（临时，安装完成后可关闭）
2. Safari 地址栏输入你的 GitHub Pages 地址，打开
3. 确认页面能正常加载（看到流水/账目/印鉴三个标签，版本号 V2.3.0）
4. 点击底部工具栏 **分享按钮**（方框箭头图标）
5. 滑动找到 **「添加到主屏幕」**，点击
6. 确认名称为「记账本」，点击右上角 **添加**
7. iPhone 桌面出现「记账本」图标

### 验证

1. **关掉翻墙工具**
2. 从桌面打开「记账本」图标
3. 应该能全屏打开，正常使用
4. 进入「印鉴」页面最底部，应显示 `V2.3.0`

## 三、日常使用

### 记账

1. 打开桌面「记账本」
2. 点击右下角红色「记」按钮
3. 选择日期 → 事由 → 银票 → 输入金额 → 附注（可选）→ 入账

### 搜索历史记录

流水页顶部搜索框输入关键字（备注、金额、银行名、事由名），跨全部历史记录搜索。

### 查看统计

账目页可按月查看公账/私账汇总和事由分布饼图。

### 数据安全

- 数据存在手机本地，不上传任何服务器
- 建议定期进入「印鉴」页面点击「导出留底」，JSON 文件存到手机（微信收藏、iCloud 均可）
- 如果换了手机或误删了 PWA，可以用「还原留底」恢复数据

---

## 四、更新到最新版本

当代码仓库有更新时：

1. **在自己 Fork 的仓库**中点击 **Sync fork** → **Update branch**（GitHub 网页操作）
2. 等待 Actions 重新部署完成（约 1-2 分钟）
3. iPhone 上**开翻墙**，打开桌面「记账本」一次
4. Service Worker 自动拉取新版本
5. **关翻墙**，进入印鉴页确认版本号已更新
6. 之后照常离线使用

> 更新不会清除你的记账数据。

## 五、常见问题

**Q: 打开应用显示空白页？**

A: 可能是 GitHub Pages 网络问题。开启翻墙重新打开一次，加载完成后即可关闭翻墙。

**Q: 数据会丢吗？**

A: 只要不删除桌面 PWA 图标、不清除 Safari 网站数据，数据就一直保留。建议定期「导出留底」。

**Q: 能从 Android 手机用吗？**

A: 可以。Chrome 浏览器打开地址 → 菜单 → 添加到主屏幕。流程与 iPhone 类似。

**Q: 能自定义事由和银行卡吗？**

A: 可以。进入「印鉴」页面，自由添加/删除事由和银票。

**Q: 如何部署到自己域名？**

A: 在 GitHub Pages 设置中绑定自定义域名即可。但需注意国内访问需备案。
